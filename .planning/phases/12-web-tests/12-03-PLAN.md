**Step 4: Create web/e2e/auth-project.spec.ts (WTEST-04)**

This E2E test uses `page.route()` to intercept ALL external calls. No real Supabase or API needed.

Setup pattern (in beforeEach):
1. Set fake auth cookie via `page.context().addCookies([{ name: "sb-test-auth-token", value: FAKE_JWT, domain: "localhost", path: "/" }])` where FAKE_JWT is a base64url-encoded JWT with `{ sub: "user-1", email: "e2e@test.com", exp: future_timestamp }`.
2. Intercept Supabase auth endpoints with `page.route("**/auth/v1/**", ...)` returning mock user.
3. Intercept `**/rest/v1/cs_projects**` (Supabase direct queries) returning mock project array.
4. Intercept `**/api/projects` returning mock project list.

Test flow:
1. Navigate to `/login`
2. Find email and password inputs using actual selectors from login page (read page.tsx first)
3. Fill email "e2e@test.com" and password "TestPassword123!"
4. Click submit button
5. Expect URL to change away from /login
6. Navigate to `/projects`
7. Expect mock project name to be visible on page
8. If add-project form exists: fill name, submit, verify new project appears

CRITICAL: The executor MUST read the actual login and projects page.tsx files to find real selectors. Use `page.getByRole()`, `page.getByPlaceholder()`, `page.getByText()` based on what exists in the components.

**Step 5: Create web/e2e/chat.spec.ts (WTEST-05)**

Setup pattern (in beforeEach):
1. Set fake auth cookie (same as auth-project spec)
2. Intercept Supabase auth endpoints
3. Intercept `**/api/chat` POST returning mock text response:
```typescript
await page.route("**/api/chat", async (route) => {
  await route.fulfill({
    status: 200,
    contentType: "text/plain; charset=utf-8",
    body: "I can help you with construction project management.",
  });
});
```

Test flow:
1. Navigate to `/ai`
2. Find chat input (textarea, input, or custom element -- read ai/page.tsx first)
3. Type "How do I create a new project?"
4. Click send button or press Enter
5. Wait for mock response text to appear on page
6. Assert text containing "construction project management" is visible

CRITICAL: The executor MUST read the actual ai/page.tsx to find real selectors for the chat input, send button, and message display area. If the page uses `useChat` from the AI SDK, the mock response format may need to match the SDK's streaming expectations -- check how the component processes responses.

NOTE: If the AI page renders messages client-side and expects a specific streaming format, adjust the mock to return the expected format. The simplest approach is plain text since the route uses `result.toTextStreamResponse()`.
