## Decisions Made

1. **Composite reuses rather than inlines `signOut()`.** Preserves the existing Keychain cleanup contract used by `restoreSession()` and any future callers; minimizes churn; protects against accidentally divergent cleanup logic.
2. **Call order: Supabase first, then profile store.** Matches RESEARCH §Fix 3 and is defensive against future throws — if `signOut()` ever throws (today it does not), a subsequent `signOutEverywhere()` call would still run the profile-store clear on its retry.
3. **iPhone 17 simulator substituted for iPhone 16.** The plan's `<verify>` block specifies `platform=iOS Simulator,name=iPhone 16`, but the local Xcode installation does not have that simulator variant (available: iPhone 16e, iPhone 17, iPhone 17 Pro/Pro Max, iPhone Air). iPhone 17 satisfies the same semantic verification (iOS Simulator app build against iOS 26.2 SDK). This is a minor environment substitution, not a contract change.
4. **Compile-only verification retained for Test D.** The plan's `<verify>` block for Task 3 specifies `xcodebuild test -only-testing:...`, but pre-existing `ready_player_8Tests.swift` async/concurrency errors (documented in `.planning/phases/29.1-fix-critical-auth-bug/deferred-items.md` and originally in Phase 22's deferred-items, predating Phase 29.1 in commit `9805e17`) still block any test-target compile. Prior-wave context explicitly instructed: "For this plan, verification should be compile-only (build-for-testing) unless the plan's `<verify>` block specifies otherwise. Do not try to fix the pre-existing async errors — they are out of scope." Followed that directive; AuthGateTests.swift itself compiles clean (not in the failed-commands list from `xcodebuild build-for-testing`).
