```typescript
export async function fetchTablePaginated<T>(
  table: string,
  options?: {
    select?: string;
    order?: { column: string; ascending?: boolean };
    eq?: { column: string; value: string | number | boolean };
    page?: number;
    pageSize?: number;
  }
): Promise<{ data: T[]; hasMore: boolean; total: number }> {
```

   Implementation:
   - Default `page = 0`, `pageSize = 25`
   - Compute `from = page * pageSize` and `to = from + pageSize - 1`
   - Use `supabase.from(table).select(options?.select || "*", { count: "exact" })`
   - Apply `.eq()`, `.order()` from options as the existing fetchTable does
   - Apply `.range(from, to)` for pagination
   - Return `{ data: (data as T[]) || [], hasMore: (count ?? 0) > to + 1, total: count ?? 0 }`
   - On error, return `{ data: [], hasMore: false, total: 0 }`

   - If any single table has count > 1000, return `NextResponse.json({ error: "Export too large. One or more tables exceed 1000 rows. Use the API with pagination for large datasets." }, { status: 413 })`

   - Keep the existing `.limit(1000)` as a safety net even when count passes
