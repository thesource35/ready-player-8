2. INSERT...SELECT backfill for existing documents (per D-06):
```sql
INSERT INTO cs_activity_events (project_id, entity_type, entity_id, action, category, actor_id, payload, created_at)
SELECT
  CASE
    WHEN da.entity_type = 'project' THEN da.entity_id
    ELSE NULL -- non-project entities may not have resolved project_id; skip complex lookup for backfill
  END AS project_id,
  'cs_documents' AS entity_type,
  d.id AS entity_id,
  'insert' AS action,
  'document' AS category,
  d.uploaded_by AS actor_id,
  to_jsonb(d) || jsonb_build_object('detail', 'document_uploaded', 'historical', true, 'filename', d.filename) AS payload,
  d.created_at
FROM cs_documents d
JOIN cs_document_attachments da ON da.document_id = d.id
WHERE d.is_current = true
  AND NOT EXISTS (
    SELECT 1 FROM cs_activity_events ae
    WHERE ae.entity_type = 'cs_documents' AND ae.entity_id = d.id
  );
```
- Uses `d.created_at` as the event timestamp (per D-06) so historical events appear at their original time.
- Includes `'historical': true` in payload so UI can distinguish backfilled events.
- `NOT EXISTS` guard prevents duplicate backfill on re-run.
- Only backfills `is_current = true` documents (one event per version chain, not per historical version).
