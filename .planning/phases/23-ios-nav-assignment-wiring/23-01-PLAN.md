# Plan 23-01: DailyCrewView Project Picker

**Replace the `mockProjects.first?.id.uuidString ?? ""` stub wired by quick task 260414-n4w with a real project picker inside DailyCrewView.**

## Why This Matters

The NavTab-to-DailyCrewView wiring (INT-03) is closed mechanically, but the view currently ships with a hardcoded first-mock-project id. A user with zero projects configured sees an empty string projectId; a user with multiple projects can only save crew for whichever happens to be first in the mock array. TEAM-05 ("User can create a daily crew assignment per project") is not meaningfully satisfied without picker UX.

## Scope

### In

1. Add `SupabaseProject` loading inside `DailyCrewView` (mirror ProjectsView.swift:6-18 hybrid pattern)
2. Add a `Menu` picker at the top of the view body, above the date picker
3. Replace the `let projectId: String` parameter with `@State private var selectedProjectId: String?` + `@AppStorage("ConstructOS.Team.LastDailyCrewProjectId")`
4. Change `ContentView.swift:745` call-site from `DailyCrewView(projectId: mockProjects.first?.id.uuidString ?? "")` to `DailyCrewView()`
5. Wire picker change to refresh crew list (same pattern as existing date `.onChange`)
6. Handle empty-project state with clear CTA to Projects tab

### Out

- iOS XCTest coverage — Phase 28
- Inline project creation from the picker
- Cross-project crew copying
- Visual/UX polish beyond matching existing Menu pattern

## Tasks

### T-23-01-01: Add project state + loader to DailyCrewView

In `ready player 8/DailyCrewView.swift`:

1. Add `@EnvironmentObject var supabase: SupabaseService` (confirm it's injected at app level; if not, use `@StateObject private var supabase = SupabaseService.shared`)
2. Add state:
   ```swift
   @State private var projects: [SupabaseProject] = []
   @AppStorage("ConstructOS.Team.LastDailyCrewProjectId") private var selectedProjectId: String = ""
   ```
3. Add computed property:
   ```swift
   private var displayProjects: [SupabaseProject] {
       supabase.isConfigured && !projects.isEmpty ? projects : mockSupabaseProjects
   }
   private var selectedProject: SupabaseProject? {
       displayProjects.first { $0.id.uuidString == selectedProjectId }
   }
   ```
4. Add `loadProjects()` async function mirroring ProjectsView pattern:
   - If `supabase.isConfigured`: `try await supabase.fetch("cs_projects", query: "order=created_at.desc")`
   - Else: projects stays empty, computed displayProjects falls back to mock
5. Add `.task { await loadProjects(); if selectedProjectId.isEmpty { selectedProjectId = displayProjects.first?.id.uuidString ?? "" }; await loadCrew() }` modifier

### T-23-01-03: Add picker UI above date

Insert this Menu block immediately below the `Text("DAILY CREW")` header and before the `DatePicker`:

```swift
Menu {
    ForEach(displayProjects) { p in
        Button(action: {
            selectedProjectId = p.id.uuidString
            Task { await loadCrew() }

        }
    }

    HStack {
        Text(selectedProject?.name ?? "Select project…")

}
```
