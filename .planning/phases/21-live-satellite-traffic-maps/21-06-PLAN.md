**Create `web/src/app/api/portal/[slug]/map/route.ts`** -- GET endpoint:
- Accept `slug` from params
- Look up portal config via service-role Supabase client (portal is public, no user auth)
- Extract `sections_config.map_overlays` from the config
- If `map_overlays.show_map` is false, return `{ data: null, message: "Map not enabled for this portal" }`
- Fetch map data based on enabled overlays:
  - If `map_overlays.equipment`: query `cs_equipment_latest_positions` filtered by `assigned_project = config.project_id`
  - If `map_overlays.photos`: query `cs_documents` with GPS coords filtered by project
  - Always: query `cs_projects` for the project's lat/lng (map center)
- Return `{ data: { overlays: map_overlays, site: { lat, lng, name }, equipment: [...], photos: [...] } }`
- Use service-role client (not authenticated) since portal pages are public
- console.error with `[portal/map]` prefix on failure
