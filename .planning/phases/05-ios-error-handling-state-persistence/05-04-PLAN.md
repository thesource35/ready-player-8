**STATE-04 — Persist messages via AppStorageJSON:**

1. First, check if AIMessage conforms to Codable. It currently has: id (UUID), role (AIRole enum), content (String), timestamp (Date). The AIRole enum cases are `user` and `assistant`. Add Codable and Equatable conformance if not present:

```swift
struct AIMessage: Identifiable, Codable, Equatable {
    let id: UUID  // Change from computed `let id = UUID()` to stored property
    let role: AIRole
    let content: String
    let timestamp: Date

    enum AIRole: String, Codable { case user, assistant }

    // Make id stored but still auto-generated
    init(role: AIRole, content: String, timestamp: Date = Date()) {
        self.id = UUID()
        self.role = role
        self.content = content
        self.timestamp = timestamp
    }

    var timestampLabel: String {
        let f = DateFormatter(); f.dateFormat = "h:mm a"; return f.string(from: timestamp)
    }

    // Equatable — compare by id
    static func == (lhs: AIMessage, rhs: AIMessage) -> Bool {
        lhs.id == rhs.id && lhs.content == rhs.content
    }
}
```

**Important:** The `let id = UUID()` pattern creates a new UUID on every decode, breaking Codable round-trips. Change to a stored property initialized via init().

2. Persist messages using session-specific key:
```swift
// Before: @State private var messages: [AIMessage] = []
// After:
@State private var messages: [AIMessage] = []

// In .task or .onAppear, load from local storage first:
// Use sessionID in the key so different sessions have different local caches
func loadLocalMessages() {
    let key = "ConstructOS.AngelicAI.Messages.\(sessionID)"
    messages = loadJSON(key, default: [AIMessage]())
}
```

Note: Can't use loadJSON in the @State initializer because sessionID is an @AppStorage property that may not be available at struct init time. Instead, load in `.onAppear` or `.task`.

3. Save on every message change:
```swift
.onChange(of: messages) { _, newValue in
    let key = "ConstructOS.AngelicAI.Messages.\(sessionID)"
    saveJSON(key, value: newValue)
}
```
