1. `parseBudgetString(budget: string): number` -- strip "$", commas, spaces. Return 0 for empty/N/A/TBD/---. parseFloat the rest, return 0 if NaN. (Per Research Pitfall 4: budget fields are strings)

2. `computeHealthScore(sections: { budget?: { spent: number; total: number } | null; schedule?: { delayedMilestones: number; totalMilestones: number } | null; issues?: { criticalOpen: number } | null }): HealthScore` -- Start at 100. Budget factor 40% weight: >90% spent = -40, >70% = -20. Schedule factor 35% weight: >30% delayed = -35, >10% = -15. Issues factor 25% weight: >3 critical = -25, >0 = -10. Map score to color using HEALTH_THRESHOLDS from constants.ts (>=80 green, >=60 yellow, else red). Map color to label using HEALTH_LABELS.

3. `computeBudgetSummary(projects: Project[], contracts: Contract[], changeOrders: ChangeOrder[]): BudgetSection` -- Sum parseBudgetString(project.budget) for contractValue. Sum parseBudgetString(contract.budget) for totalBilled (use contract stage to determine billed vs remaining). Calculate changeOrderNet as sum of CO amounts where status is "approved". Retainage = contractValue * 0.10 default. percentComplete = totalBilled / contractValue * 100 (clamped 0-100 per D-56e).

4. `computeScheduleSummary(tasks: Array<{ id: string; name: string; progress: number; status: string }>): ScheduleSection` -- Map tasks to milestones with percentComplete. Count delayed = tasks where status contains "delayed" or "overdue". Top 5-8 milestones per D-26.

5. `computeIssuesSummary(rfis: Rfi[], changeOrders: ChangeOrder[]): IssuesSection` -- Count criticalOpen = RFIs where priority is "critical" or "urgent" and status is not "closed". totalOpen = RFIs + COs where status is not "closed"/"approved".

6. `computeSafetySummary(incidents: Array<{ id: string; title: string; severity: string; created_at: string }>): SafetySection` -- Count by severity (minor/moderate/serious). daysSinceLastIncident = diff between now and most recent incident created_at, or null if no incidents. Per D-16.

7. `computeTeamSummary(assignments: ProjectAssignment[], members: TeamMember[], activity: ActivityEvent[]): TeamSection` -- totalMembers from unique member_ids in assignments. roleBreakdown from member.role counts. recentActivity = first 5 activity items sorted by created_at desc. Per D-14.

8. `computeFeatureCoverage(report: { budget: { data: BudgetSection | null }; schedule: { data: ScheduleSection | null }; issues: { data: IssuesSection | null }; safety: { data: SafetySection | null }; team: { data: TeamSection | null }; documents: { data: DocumentPhotoSection | null } }): { active: number; total: number }` -- Count sections where data is not null and has meaningful non-zero values. total = 6 always (FEATURE_COUNT_TOTAL). Per D-16c.

9. `computePortfolioRollup(projectReports: ProjectReport[]): PortfolioRollup` -- Aggregate across all project reports. Sum contractValue, totalBilled, changeOrderNet. Compute aggregateHealth from averaged scores. Build PortfolioProject[] array with featureCoverage per project.

Run tests -- they must all PASS.
