**Maps page (web/src/app/maps/page.tsx):**

Currently the maps page checks for the token inside the useEffect on line 50 and silently returns if missing. The map container div renders but stays empty.

1. Add a state variable to track whether the token exists: read `process.env.NEXT_PUBLIC_MAPBOX_TOKEN` at the component level (outside useEffect) into a const `token`.
2. The token is already read on line 50 inside useEffect. Add a check BEFORE the map container div in the JSX. If `!token`, render the "Maps Unavailable" fallback card INSTEAD of the map container div (not in addition to it).
3. The fallback card must use these exact styles and copy:
   - Container: `borderRadius: 16, padding: 32, textAlign: "center", background: "var(--surface)", border: "1px solid rgba(51,84,94,0.2)"`
   - Emoji: fontSize 48, marginBottom 8 (use a globe or map-pin emoji)
   - Heading: fontSize 20, fontWeight 900, marginBottom 8, text "Maps Unavailable"
   - Body: fontSize 13, color "var(--muted)", text "Configure MAPBOX_TOKEN to enable interactive maps."
4. The rest of the page (sites list, satellites, routes, overlays) should still render — only the map container area gets the fallback.

**Jobs page (web/src/app/jobs/page.tsx):**

The jobs page fetches from /api/jobs and stores results in state. It already has a filtered empty state at approximately line 373 that says "No jobs in this filter yet". This needs a SEPARATE check for when the total jobs array (before filtering) is empty.

1. Read the full jobs/page.tsx first to understand the data flow. Jobs are fetched in useEffect and stored in a `jobs` state variable. `filteredJobs` is derived via useMemo.
2. Add a check: when `jobs.length === 0` AND loading is complete (not currently loading), show a global "No Jobs Found" empty state. This is DIFFERENT from the existing filtered empty state.
3. Place this check in the JSX where the job listings are rendered. If `jobs.length === 0` (and not loading), show:
   - Container: `borderRadius: 16, padding: 32, textAlign: "center", background: "var(--surface)", border: "1px solid rgba(51,84,94,0.2)"`
   - Emoji: fontSize 48, marginBottom 8 (use a briefcase or clipboard emoji)
   - Heading: fontSize 20, fontWeight 900, marginBottom 8, text "No Jobs Found"
   - Body: fontSize 13, color "var(--muted)", text "No job listings are available right now. Check back soon for new opportunities."
4. The EXISTING filtered empty state ("No jobs in this filter yet") at ~line 373 should remain for when `jobs.length > 0` but `filteredJobs.length === 0`.
