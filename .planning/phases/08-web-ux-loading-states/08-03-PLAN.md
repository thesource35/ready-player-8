**AngelicPromptToggle SSR fix (web/src/app/components/AngelicPromptToggle.tsx):**

The useState initializer on line 11 accesses `window.localStorage` directly, which crashes during SSR because `window` is undefined on the server.

1. In the useState initializer function (lines 11-18), add `if (typeof window === "undefined") return true;` as the FIRST line inside the callback, BEFORE the try block.
2. The result should be:
```typescript
const [enabled, setEnabled] = useState(() => {
  if (typeof window === "undefined") return true;
  try {
    const storedValue = window.localStorage.getItem(ANGELIC_BUBBLE_ENABLED_KEY);
    return storedValue === null ? true : storedValue === "true";
  } catch {
    return true;
  }
});
```
3. Do NOT change anything else in the file. The useEffect on lines 20-29 is already safe (useEffect only runs on client). The event handlers are also safe.
