<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated user -> cs_video_sources/cs_video_assets | RLS enforces org membership via user_orgs; writes further role-gated (D-39) |
| service_role -> all Phase 22 tables | Bypasses RLS; used by ffmpeg worker, retention cron, webhook handler — must be guarded at infrastructure layer (env var scoping) |
| anonymous -> 'videos' storage bucket | Bucket is private; no anonymous read path; portal access flows through server-minted signed URLs only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01-01 | Information Disclosure | cs_video_sources cross-org SELECT | mitigate | RLS policy `cs_video_sources_select` scopes by `user_orgs where user_id = auth.uid()` — same shape as Phase 21 cs_equipment (D-16) |
| T-22-01-02 | Elevation of Privilege | cs_video_sources DELETE by non-admin | mitigate | Delete policy requires `role in ('owner','admin')` per D-39 |
| T-22-01-03 | Elevation of Privilege | cs_video_assets DELETE by non-creator non-admin | mitigate | Delete policy requires `created_by = auth.uid() OR role in ('owner','admin')` per D-39 |
| T-22-01-04 | Spoofing | storage bucket cross-org path access | mitigate | `(storage.foldername(name))[1]::uuid in user_orgs` policy — attacker cannot guess org_id UUIDs, and even if they did, RLS blocks |
| T-22-01-05 | Information Disclosure | cs_video_webhook_events leak via user query | accept | Service-role-only table; authenticated users have no policy granting access; default-deny applies |
| T-22-01-06 | Tampering | Raw-upload MIME spoofing at storage insert | mitigate | `allowed_mime_types` on bucket limits to mp4/quicktime/m4v; ffmpeg worker re-validates codec server-side (D-31) |
| T-22-01-07 | DoS | 2GB upload limit | mitigate | `file_size_limit: 2147483648` on bucket; client-side pre-check (D-31); server-side enforced by Supabase Storage |
| T-22-01-08 | Repudiation | Orphaned Mux live_input if DB insert rolls back | mitigate | API-route plan 22-03 implements compensating delete on failure; this is a note for the writer plan, not schema |
</threat_model>
