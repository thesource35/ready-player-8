<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous portal viewer → /api/portal/video/* | Unauthenticated; portal_token is the only credential; per-request scope check |
| Portal viewer's browser → Mux/Supabase Storage | JWT/signed URL bound to specific playback_id/asset; 5-min live TTL, 1h VOD TTL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-22-09-01 | Spoofing | Portal token for project A used to access project B's assets | mitigate | Portal-token route cross-checks `asset.project_id === link.project_id` (RESEARCH pseudocode); returns 403 if not |
| T-22-09-02 | Elevation | Portal accesses drone-typed asset | mitigate | `if asset.source_type === 'drone' return 403` (D-22) — NO code path around this check |
| T-22-09-03 | Elevation | Portal accesses non-shared VOD | mitigate | `if kind === 'vod' && !portal_visible return 403` |
| T-22-09-04 | Elevation | Portal DVR-scrubs a live stream | mitigate | Live playback route mints a JWT bound to playback_id; PORTAL player passed `targetLiveWindow={0}` (web) / `requiresLinearPlayback=true` (iOS); server also can set `playback_restriction_id` on the JWT tied to a Mux Playback Restriction forcing LIVE-only (explore during impl — optional hardening) |
| T-22-09-05 | Information Disclosure | Portal download of VOD segment | mitigate | No download affordance in portal VideoClipPlayer; Cache-Control: no-store on manifest; signed URL 1h TTL; portal player does not expose save/share |
| T-22-09-06 | DoS | Portal token reuse floods mint endpoint | mitigate | Rate limit 30 req/min/IP (D-37); 429 + Retry-After |
| T-22-09-07 | Repudiation | No audit trail of portal video views | accept-for-v1 | Phase 20's existing portal analytics captures page views; D-40 emits `portal_video_view` event; if deeper audit needed → follow-up |
| T-22-09-08 | Spoofing | Expired/revoked portal token allows playback | mitigate | Route checks `is_revoked=false` AND `expires_at > now()` before any mint |
</threat_model>
