<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API route | User messages cross from untrusted browser to server |
| API route -> Supabase | Authenticated queries via RLS-protected tables |
| API route -> Anthropic | Tool results sent as context to LLM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | tool input params | mitigate | Zod inputSchema validates all tool parameters before execute runs; no raw SQL interpolation |
| T-18-02 | Information Disclosure | tool results | mitigate | RLS policies on cs_projects/cs_contracts ensure users only see their own data; .limit(20) prevents bulk extraction |
| T-18-03 | Elevation of Privilege | generate_rfi / draft_change_order | mitigate | Tools return DRAFT JSON only -- no database insert in tool execute; _action: "review_before_saving" signals UI |
| T-18-04 | Spoofing | unauthenticated tool access | mitigate | Each data tool checks for null supabase client and returns error; CSRF check on route entry |
| T-18-05 | Tampering | prompt injection via tool results | accept | Tool results are structured JSON from Supabase RLS-filtered queries; low risk of injection since data is user's own |
| T-18-06 | Denial of Service | excessive tool rounds | mitigate | stopWhen: stepCountIs(5) limits tool loop to max 5 rounds |
</threat_model>
