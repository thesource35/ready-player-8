**OperationsCore.swift — 2 force unwraps (lines 502, 503):**

Both are `filterType!` and `filterStatus!` inside a closure that checks `filterType == nil` first. Replace with safe pattern using optional map or optional chaining:

Replace the entire `filtered` computed property (lines 500-505):
```swift
private var filtered: [SafetyIncident] {
    incidents.filter { incident in
        (filterType == nil || incident.type == filterType) &&
        (filterStatus == nil || incident.status == filterStatus)
    }
}
```

Note: Comparing `incident.type == filterType` where filterType is `IncidentType?` works because Swift's `==` operator handles optional comparison — if filterType is nil, it returns false (but that branch is already guarded by `filterType == nil ||`). Actually, the simpler approach that avoids any subtlety:

```swift
private var filtered: [SafetyIncident] {
    incidents.filter { incident in
        (filterType.map { incident.type == $0 } ?? true) &&
        (filterStatus.map { incident.status == $0 } ?? true)
    }
}
```

This reads as: "if filterType has a value, check equality; if nil, pass through (true)."
