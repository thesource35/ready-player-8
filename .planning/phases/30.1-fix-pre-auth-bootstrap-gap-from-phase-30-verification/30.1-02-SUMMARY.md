## Fresh-Install Bootstrap Flow (end-to-end)

1. **Launch (fresh install / wiped device)** → `AuthGateView.loginView` renders. `supabase.isConfigured == false`, so the gear-icon "Configure Backend" button + Theme.muted helper copy appear below the "New to ConstructionOS? Create an account" HStack.

2. **Tap "Configure Backend"** → `showBackendConfig = true` flips. The `.sheet` modifier on the body ZStack presents `BackendConfigSheet` (built in Plan 30.1-01). The sheet receives an explicit `.environmentObject(supabase)` (defensive against iOS environment-inheritance variance).

3. **User enters Base URL + anon key + taps SAVE** → Plan 30.1-01's Save handler trims, validates (T-30.1-04 https/localhost gate), runs `classifyKeyPrefix` (T-30.1-02 service-role warning), then calls `supabase.configure(baseURL:apiKey:)`. That call writes to Keychain (SEC-02/SEC-03 invariant) and ends with `objectWillChange.send()`.

4. **Sheet dismisses + UI re-renders on the same pass** → `objectWillChange.send()` re-evaluates the `if !supabase.isConfigured` branch in `loginView`. The button is replaced by the green checkmark "Backend configured" indicator. **No app restart.**

5. **User taps SIGN IN or CREATE ACCOUNT** → the existing handler (byte-identical to before this phase) calls `supabase.signIn(...)` / `supabase.signUp(...)`. Previously this threw `SupabaseError.notConfigured` ("Supabase not configured. Enter your Base URL and API key in COMMAND → Integration Hub." — `SupabaseService.swift` line 124). Now `isConfigured == true` so the call proceeds normally and authentication completes.

The chicken-and-egg lockout from Phase 30 NOTIF-05 UAT is closed.
