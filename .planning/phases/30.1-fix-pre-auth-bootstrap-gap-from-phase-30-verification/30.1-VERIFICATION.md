## Phase Scope Context

Phase 30 NOTIF-05 UAT (2026-04-24) surfaced a chicken-and-egg lockout: a fresh-install or wiped-device user lands on `AuthGateView` and cannot authenticate because Supabase is not configured, but the only configuration UI (Integration Hub) lives behind the auth gate. The error message thrown by `SupabaseService` ("Supabase not configured. Enter your Base URL and API key in COMMAND → Integration Hub.") points the user at a screen they cannot reach.

This phase closes backlog 999.3 with a 3-plan structure:

- **Plan 30.1-01** (Wave 1): `BackendConfigSheet` view + `validateBaseURL` + `classifyKeyPrefix` helpers in `ContentView.swift`; pure-logic XCTest scaffold in `ready player 8Tests/BackendConfigSheetTests.swift`. See `30.1-01-SUMMARY.md`.
- **Plan 30.1-02** (Wave 2): `AuthGateView.loginView` wiring — `@State private var showBackendConfig`, conditional "Configure Backend" / "Backend configured ✓" affordance, `.sheet(isPresented:)` modifier on the body ZStack. See `30.1-02-SUMMARY.md`.
- **Plan 30.1-03** (Wave 3, this report): VERIFICATION + UAT-LOG + ROADMAP/REQUIREMENTS reconciliation.

All `grep` / `xcodebuild` commands below were run at execution time (2026-04-24) against the working tree. Outputs are pasted verbatim. The `xcodebuild test` row calls out the pre-existing `ready_player_8Tests.swift` async/concurrency blocker (Phase 22 / 29.1 / 30 precedent — compile-only verification substituted).
