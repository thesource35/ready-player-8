  <action>
For each of the three pages, apply the same pattern:

1. Add a `fetchError` state variable:
   ```typescript
   const [fetchError, setFetchError] = useState<string | null>(null);
   ```

2. Update the `.catch()` handler in the useEffect to set user-visible error state instead of just console.error:
   ```typescript
   .catch(() => {
     setFetchError("Could not load live data -- showing cached results. Check your connection and refresh.");
   });
   ```
   Keep console.error as well for debugging: `console.error("[Fetch] Load failed");`

3. Render the error banner at the top of the page content (inside the main container div, before the first content section). Use inline styles consistent with the project's design token pattern:
   ```tsx
   {fetchError && (
     <div style={{
       background: "rgba(217, 77, 72, 0.15)",
       border: "1px solid var(--red)",
       borderRadius: 10,
       padding: "12px 16px",
       marginBottom: 16,
       display: "flex",
       justifyContent: "space-between",
       alignItems: "center",
       fontSize: 13,
       color: "var(--red)",
     }}>
       <span>{fetchError}</span>
       <button
         onClick={() => setFetchError(null)}
         style={{
           background: "none",
           border: "none",
           color: "var(--red)",
           cursor: "pointer",
           fontSize: 16,
           padding: "0 4px",
         }}
         aria-label="Dismiss error"
       >
         x
       </button>
     </div>
   )}
   ```

4. Also handle non-ok response status in the `.then()` chain. Before parsing JSON, check `res.ok`:
   ```typescript
   .then(res => {
     if (!res.ok) throw new Error(`HTTP ${res.status}`);
     return res.json();
   })
   ```
   This ensures HTTP errors (like 429, 500) also trigger the error banner.
