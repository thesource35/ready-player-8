# Requirements: ConstructionOS Production Hardening

**Defined:** 2026-04-04
**Core Value:** Every user action must either succeed visibly or fail with a clear, actionable message — no silent data loss, no undetected errors, no security gaps.

## v1 Requirements

### Security — Secrets & Credentials

- [ ] **SEC-01**: All API keys (Anthropic, Paddle, Coinbase) stored in iOS Keychain, not UserDefaults
- [ ] **SEC-02**: Supabase credentials (BaseURL, ApiKey) stored in iOS Keychain, not UserDefaults
- [ ] **SEC-03**: Migration path: read Keychain first, fall back to UserDefaults, migrate and delete old entry
- [ ] **SEC-04**: Web environment variables validated at startup (missing keys logged, not silently ignored)
- [ ] **SEC-05**: Sanitize Mapbox popup HTML — escape user-sourced data in setHTML() calls
- [ ] **SEC-06**: Remove 'unsafe-eval' from CSP script-src (next.config.ts)
- [ ] **SEC-07**: Tighten CSP wildcards for mapbox.com to specific subdomains
- [ ] **SEC-08**: Rotate Supabase secret key (committed in .env.local — currently exposed in git history)
- [ ] **SEC-09**: Add .env*, .DS_Store, xcuserdata/, DerivedData/, *.pem to root .gitignore
- [ ] **SEC-10**: Remove `@next/swc-darwin-arm64` from package.json dependencies (breaks Linux CI)
- [ ] **SEC-11**: Remove orphaned `untitled folder/` and `web/src/.next/` from repo
- [ ] **SEC-12**: Square webhook signature bypass — don't default to empty string when env var missing; reject unsigned requests

### Security — Authentication

- [ ] **AUTH-01**: Supabase Auth configured with email/password provider
- [ ] **AUTH-02**: iOS AuthGateView wired to Supabase Auth SDK (sign up, login, logout)
- [ ] **AUTH-03**: iOS auth tokens persisted in Keychain, auto-refreshed before expiry
- [ ] **AUTH-04**: Web auth with @supabase/ssr cookie-based sessions
- [ ] **AUTH-05**: Web auth middleware (middleware.ts) validates session on protected routes
- [ ] **AUTH-06**: Demo/DEBUG mode preserved — developers can skip auth in debug builds
- [ ] **AUTH-07**: 2FA backup code verification flow implemented (currently "coming soon")
- [ ] **AUTH-08**: 2FA TOTP validation must verify against server (currently accepts any 6-digit code except 000000)
- [ ] **AUTH-09**: SSO buttons must use real OAuth (currently fake auth with hardcoded "sso" token)
- [ ] **AUTH-10**: Forgot password must make actual API call (currently no-op that shows fake success message)
- [ ] **AUTH-11**: Leads endpoint requires rate limiting or CAPTCHA (currently no auth, open to spam)

### Security — Row-Level Security

- [ ] **RLS-01**: user_id column added to all Supabase tables
- [ ] **RLS-02**: Existing rows backfilled with admin user_id before RLS enabled
- [ ] **RLS-03**: RLS policies enforce per-user read/write on all tables
- [ ] **RLS-04**: iOS SupabaseService sends auth token with all requests
- [ ] **RLS-05**: Web Supabase client sends auth token with all requests
- [ ] **RLS-06**: Web API PATCH/DELETE routes add ownership checks (punch, tasks, projects — any user can currently modify any row)
- [ ] **RLS-07**: Jobs API should not use service role key for public GET (bypasses RLS)

### Security — Web Hardening

- [ ] **WEB-01**: Email format validation on /api/leads endpoint
- [ ] **WEB-02**: CSRF protection on all POST endpoints
- [ ] **WEB-03**: API keys not exposed in client-side code or fallback responses
- [ ] **WEB-04**: Paddle webhook signature verification
- [ ] **WEB-05**: Paddle webhook handler updates subscription status in Supabase

### Crash Safety — iOS

- [ ] **CRASH-01**: Replace force unwraps in SupabaseService.swift (lines 199, 221, 464, 702) with guard-let
- [ ] **CRASH-02**: Replace force unwraps in ContentView.swift footer URLs (lines 93, 95, 97) with guard-let
- [ ] **CRASH-03**: Replace fatalError in PersistenceController.swift:42 with graceful fallback
- [ ] **CRASH-04**: Guard force unwraps in OperationsField.swift (lines 709, 712) and OperationsCore.swift (502, 503)
- [ ] **CRASH-05**: Guard SecurityAccessView.swift:204 hash.last! safely

### Error Handling — iOS

- [ ] **ERR-01**: All empty catch blocks replaced with CrashReporter.shared.log() + appropriate user alert
- [ ] **ERR-02**: All silent try? calls replaced with do-catch blocks (20+ instances across SupabaseService, UIHelpers, views)
- [ ] **ERR-03**: Error categorization: expected/recoverable (log) vs unexpected (alert) vs non-critical (DEBUG only)
- [ ] **ERR-04**: Chat API fallback clearly indicates "mock data" vs "live data"
- [ ] **ERR-05**: Remove 4 unguarded print() calls that leak info in production (AngelicAIView, FinancialInfrastructure, PersistenceController)
- [ ] **ERR-06**: Use .textInputAutocapitalization(.never) instead of deprecated .autocapitalization(.none)

### Error Handling — Web

- [ ] **WERR-01**: All API routes return structured error JSON
- [ ] **WERR-02**: Chat route distinguishes "API unavailable" vs "temporary error"
- [ ] **WERR-03**: Missing env vars detected and logged at startup
- [ ] **WERR-04**: Replace empty .catch() console.error in tasks/punch/ops pages with user-visible error
- [ ] **WERR-05**: Jobs API validates response body shape before using
- [ ] **WERR-06**: Wrap req.json() in try/catch on all 8 API routes missing it (projects, contracts, punch, feed, tasks POST/PATCH/DELETE, paddle webhook)
- [ ] **WERR-07**: Export API returns 503 when Supabase not configured (not fake data with 200)
- [ ] **WERR-08**: POST routes return 201 for successful creation (not 200) — standardize across all routes
- [ ] **WERR-09**: DELETE routes return proper error status when deletion fails (not 200 with success:false)

### Error Boundaries & Loading States — Web

- [ ] **UX-01**: Add error.tsx boundaries to all 22 unprotected pages
- [ ] **UX-02**: Mapbox token missing = show "Maps unavailable" message
- [ ] **UX-03**: Jobs page shows "No results" empty state
- [ ] **UX-04**: AI page shows loading indicator during initial load
- [ ] **UX-05**: Punch/ops/tasks pages show loading indicators during fetch (not silent swap)

### Form Validation — Web

- [ ] **VAL-02**: Profile page email validation uses proper regex
- [ ] **VAL-03**: Jobs page contactEmail validated before submission
- [ ] **VAL-04**: Text inputs have maxLength attributes matching API limits

- [ ] **VAL-06**: Rental form does NOT show success when API call fails (fix race condition)
- [ ] **VAL-07**: addProject() on projects page must persist to database (currently local-state only)
- [ ] **VAL-08**: Projects page add form shows validation feedback (not silent return)

### Accessibility
